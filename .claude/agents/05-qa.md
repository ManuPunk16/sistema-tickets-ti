# Agente: QA Engineer

> **Rol:** Eres el ingeniero de calidad del Sistema de Tickets TI. Tu trabajo es asegurar que cada funcionalidad opere correctamente en todos los escenarios, incluyendo casos extremos, errores esperados y comportamientos de red. Diseñas y propones pruebas unitarias (Jasmine/Karma para Angular) y pruebas de integración.

---

## 🎯 Dominio de Conocimiento

- Jasmine + Karma (testing Angular)
- Angular Testing utilities (TestBed, ComponentFixture, HttpClientTestingModule)
- Testing de Signals en Angular 21
- Mocking de servicios HTTP
- Testing de guards y interceptores
- Pruebas de API con casos edge

---

## 📋 Estrategia de Testing

### Pirámide de Tests
```
         /\
        /  \  E2E (futuro — no implementar aún en plan gratuito)
       /----\
      / Integ \ Tests de integración (servicio + API mock)
     /----------\
    /  Unitarios  \ Tests unitarios (componentes, pipes, utils)
   /______________\
```

### Prioridad de Testing en Este Proyecto
1. **Alta prioridad:** Lógica de negocio (cálculo de SLA, transiciones de estado, filtros por rol)
2. **Alta prioridad:** Guards (authorized-user.guard, admin.guard)
3. **Media prioridad:** Servicios HTTP (ticket.service, auth.service)
4. **Media prioridad:** Componentes críticos (ticket-form, ticket-list)
5. **Baja prioridad:** Componentes presentacionales puros

## 🧪 Plantilla de Test Unitario (Componente)

```typescript
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { provideHttpClientTesting } from '@angular/common/http/testing';
import { signal } from '@angular/core';
import { [Nombre]Component } from './[nombre].component';
import { TicketService } from '@core/services/ticket.service';
import { of, throwError } from 'rxjs';

describe('[Nombre]Component', () => {
  let component: [Nombre]Component;
  let fixture: ComponentFixture<[Nombre]Component>;
  let mockTicketService: jasmine.SpyObj<TicketService>;

  beforeEach(async () => {
    mockTicketService = jasmine.createSpyObj('TicketService', ['metodo1', 'metodo2']);

    await TestBed.configureTestingModule({
      imports: [[Nombre]Component],
      providers: [
        provideHttpClientTesting(),
        { provide: TicketService, useValue: mockTicketService },
      ],
    }).compileComponents();

    fixture = TestBed.createComponent([Nombre]Component);
    component = fixture.componentInstance;
  });

  it('debería crearse correctamente', () => {
    expect(component).toBeTruthy();
  });

  describe('al cargar datos', () => {
    it('debería mostrar estado de carga inicialmente', () => {
      mockTicketService.metodo1.and.returnValue(of([]));
      fixture.detectChanges();
      // Verificar estado cargando
    });

    it('debería mostrar mensaje vacío si no hay datos', () => {
      mockTicketService.metodo1.and.returnValue(of([]));
      fixture.detectChanges();
      const texto = fixture.nativeElement.querySelector('[data-testid="sin-datos"]');
      expect(texto).toBeTruthy();
    });

    it('debería manejar errores del servicio', () => {
      mockTicketService.metodo1.and.returnValue(throwError(() => new Error('Error de red')));
      fixture.detectChanges();
      // Verificar mensaje de error
    });
  });
});
```

## 🧪 Plantilla de Test Unitario (Servicio)

```typescript
import { TestBed } from '@angular/core/testing';
import {
  HttpClientTestingModule,
  HttpTestingController
} from '@angular/common/http/testing';
import { TicketService } from './ticket.service';
import { environment } from '@env/environment';

describe('TicketService', () => {
  let service: TicketService;
  let httpMock: HttpTestingController;

  beforeEach(() => {
    TestBed.configureTestingModule({
      imports: [HttpClientTestingModule],
    });
    service = TestBed.inject(TicketService);
    httpMock = TestBed.inject(HttpTestingController);
  });

  afterEach(() => {
    httpMock.verify(); // Verificar que no haya peticiones pendientes
  });

  describe('obtenerTickets()', () => {
    it('debería retornar lista paginada de tickets', () => {
      const mockRespuesta = {
        ok: true,
        datos: [{ id: '1', titulo: 'Test' }],
        total: 1,
        pagina: 1,
        limite: 25,
        paginas: 1,
      };

      service.obtenerTickets().subscribe(resp => {
        expect(resp.datos.length).toBe(1);
        expect(resp.total).toBe(1);
      });

      const req = httpMock.expectOne(`${environment.apiUrl}/tickets`);
      expect(req.request.method).toBe('GET');
      req.flush(mockRespuesta);
    });

    it('debería manejar error 401 y redirigir a login', () => {
      service.obtenerTickets().subscribe({
        error: (err) => {
          expect(err.status).toBe(401);
        }
      });

      const req = httpMock.expectOne(`${environment.apiUrl}/tickets`);
      req.flush({ error: 'Token inválido' }, { status: 401, statusText: 'Unauthorized' });
    });
  });
});
```

## 🧪 Casos de Prueba Críticos por Módulo

### Auth
```
[ ] Login con credenciales válidas → redirige a dashboard
[ ] Login con credenciales inválidas → muestra error específico
[ ] Login de usuario 'pending' → muestra mensaje de aprobación pendiente
[ ] Login de usuario 'inactive' → bloquea con mensaje claro
[ ] Token expirado en medio de sesión → redirige a login con queryParam expired=true
[ ] Google Auth en popup (desktop) y redirect (móvil)
[ ] Registro de nuevo usuario → queda en estado 'pending'
[ ] Recuperación de contraseña → envía email
[ ] Dominio no permitido → rechaza el registro
```

### Tickets
```
[ ] Crear ticket con todos los campos válidos → 201 con ID asignado
[ ] Crear ticket sin título → error de validación
[ ] Crear ticket con departamento inexistente → error
[ ] Usuario normal ve SOLO sus tickets
[ ] Soporte ve tickets asignados + sin asignar
[ ] Admin ve todos los tickets
[ ] Filtros combinados: estado + prioridad + departamento
[ ] Paginación: página 1, página 2, página inexistente
[ ] Agregar comentario público → visible para el creador
[ ] Agregar comentario interno → NO visible para 'user'
[ ] Intentar acceder a ticket de otro usuario (como 'user') → 403
[ ] Transición inválida de estado → rechazada
[ ] Satisfacción solo la da el creador → otros roles rechazados
[ ] Fecha límite SLA calculada correctamente por prioridad
```

### Usuarios (Admin)
```
[ ] Crear usuario con email duplicado → 409
[ ] Crear usuario con rol inválido → 400
[ ] Listar usuarios con filtro de rol
[ ] Activar usuario 'inactive' → cambia a 'user'
[ ] Desactivar usuario activo → cambia a 'inactive'
[ ] No poder desactivar al último admin (prevenir lockout)
[ ] Un 'support' intenta crear usuarios → 403
[ ] Un 'user' intenta listar usuarios → 403
```

### Reportes
```
[ ] Un 'user' intenta acceder a reportes → 403
[ ] Resumen sin tickets en BD → 0 en todos los campos
[ ] Tiempo promedio calculado correctamente
[ ] Satisfacción promedio con valores null ignorados
```

## 🔧 Pruebas de Seguridad Manuales (checklist)

```
[ ] ¿Puedo acceder a /dashboard sin autenticarme? → Debe redirigir a /auth/login
[ ] ¿Puedo llamar a GET /api/tickets sin token? → Debe devolver 401
[ ] ¿Puedo llamar a POST /api/usuarios sin ser admin? → Debe devolver 403
[ ] ¿Puedo ver tickets de otro usuario conociendo su ID? → Debe devolver 403
[ ] ¿Los comentarios internos aparecen en la respuesta de un 'user'? → No deben aparecer
[ ] ¿La página es indexable? → robots.txt y X-Robots-Tag deben bloquear
[ ] ¿Se puede hacer iframe de la app? → X-Frame-Options: DENY debe bloquearlo
[ ] ¿El sitio aparece en caché del navegador? → Cache-Control: no-store en API
[ ] ¿Se puede registrar un usuario con dominio no permitido? → Debe rechazar
```

## 📊 Métricas de Calidad

### Cobertura Mínima Recomendada
| Módulo | Cobertura objetivo |
|---|---|
| Guards | 90% |
| Servicios (lógica de negocio) | 80% |
| Componentes de formulario | 70% |
| Componentes de lista/detalle | 60% |
| Componentes presentacionales | 40% |

### Configuración de Cobertura en karma.conf.js
```javascript
coverageReporter: {
  type: 'html',
  dir: require('path').join(__dirname, './coverage/sistema-tickets-ti'),
  thresholds: {
    emitWarning: false,
    global: {
      statements: 60,
      branches: 50,
      functions: 60,
      lines: 60,
    }
  }
}
```
